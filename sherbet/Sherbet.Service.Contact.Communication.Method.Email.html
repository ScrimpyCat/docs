    add(identity, email)

    Implement the behaviour for adding a new communication method and associating it with the given identity.

    If the contact is already in use, it will return an error.

    If the operation was successful return :ok.

    Callback implementation for Sherbet.Service.Contact.Communication.Method.add/2.

    contacts(identity)

    Implement the behaviour for retrieving the contacts of the communication method for the given identity.

    If the operation was successful return { :ok, contacts }, where contacts is the list of communication methods associated with the given identity and their current verification status and priority. Otherwise return an error.

    Callback implementation for Sherbet.Service.Contact.Communication.Method.contacts/1.

    finalise_removal(email, key)

    Implement the behaviour for finalising a removal request of an unverified communication method.

    If the communication method has been verified, then it should return an error.

    If the operation was successful return :ok.

    Callback implementation for Sherbet.Service.Contact.Communication.Method.finalise_removal/2.

    finalise_verification(identity, email, key)

    Implement the behaviour for finalising a verification request of an unverified communication method.

    If the communication method has already been verified, then it should return an error.

    If the operation was successful return :ok. Otherwise return an error.

    Callback implementation for Sherbet.Service.Contact.Communication.Method.finalise_verification/3.

    If the operation was successful return { :ok, contact }, where contact is the primary communication method associated with the given identity. Otherwise return an error.

    remove(identity, email)

    Implement the behaviour for removing a communication method associated with the given identity.

    request_removal(email)

    Implement the behaviour for requesting an unverified communication method be removed. This should generate the unique key that will be sent to the contact, that the requester will require in order to finalise the removal.

    If the communication method has been verified, then it should return an error.

    If the operation was successful return :ok.

    Callback implementation for Sherbet.Service.Contact.Communication.Method.request_removal/1.

    request_verification(identity, email)

    Implement the behaviour for requesting an unverified communication method be verified. This should generate the unique key that will be sent to the contact, that the requester will require in order to finalise the verification of that contact.

    If the communication method has already been verified, then it should return an error.

    If the operation was successful return :ok. Otherwise return an error.

    Callback implementation for Sherbet.Service.Contact.Communication.Method.request_verification/2.

    set_priority(identity, email, atom)

    Implement the behaviour for setting a communication method associated with the given identity to a new priority.

    Only one communication method per identity may be set as :primary. If one already exists, change it to secondary to allow for this new one to be made :primary.

    If the operation was successful return :ok. Otherwise return the error.

    Callback implementation for Sherbet.Service.Contact.Communication.Method.set_priority/3.

    verified?(identity, email)

    Implement the behaviour for identifying if a communicaton method belonging to the given identity has been verified.

    If the operation was successful return whether it was verified or not (true if it was verified, otherwise false). Otherwise return an error.

    Callback implementation for Sherbet.Service.Contact.Communication.Method.verified?/2.
