    Gobstopper vdev Gobstopper.Service.Auth.Identity.Credential behaviour View Source

    Manages the interactions with credentials.

    Credential implementations will implement the given callbacks to handle the specific credential type.

    Implementing a credential

    Credentials should be implemented in a module conforming to Gobstopper.Service.Auth.Identity.Credential.type. Where type is the capitalized credential type.

    e.g. For a credential that should be identified using the :email atom, then the implementation for that credential should fall under Gobstopper.Service.Auth.Identity.Credential.Email.

    info(identity :: Gobstopper.Service.Auth.Identity.Model.t()) ::
  {state :: :unverified | :verified, presentable :: String.t()} |
  {:none, nil}

    Implement the behaviour for retrieving the presentable information for the credential of a given identity.

    If one exists return the state of the credential (:unverified or :verified) and the presentable string. Otherwise return { :none, nil }.

    Verification state is used to infer whether the given credential is guaranteed to be owned by the identity owner.
